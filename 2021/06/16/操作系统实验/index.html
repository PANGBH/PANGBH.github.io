实验一 进程管理

实验目的

（1）加深对于进程并发执行概念的理解。

（2）实践并发进程的创建和控制方法。

（3）观察和体验进程的动态特性。

（4）进一步理解进程生命期期间创建、变换、撤销状态变换的过程。

（5）掌握进程控制的方法，了解父子进程间的控制和协作关系。

（6）练习Linux系统中进程创建与控制有关的系统调用的编程和调试技术。

实验环境

Cygwin64

Visual Studio 2019

实验说明

与进程创建、执行有关的系统调用说明

进程可以通过系统调用fork()创建子进程并和其子进程并发执行.

子进程初始的执行映像是父进程的一个复本.

子进程可以通过exec()系统调用族装入一个新的执行程序。

父进程可以使用wait()或waitpid()系统调用等待子进程的结束并负责收集和清理子进程的退出状态。

fork()系统调用语法:

#include <unistd.h>

pid_t fork(void);

fork成功创建子进程后将返回子进程的进程号,不成功会返回-1.

getpid()—获取进程的pid

exec 系统调用

exec 系统调用有一组6个函数,其中示例实验中引用了 execve 系统调用语法:

#include <unistd.h>

int execve(const char *path, const char *argv[], const char * envp[]);

path 要装入的新的执行文件的绝对路径名字符串.

argv[] 要传递给新执行程序的完整的命令参数列表(可以为空).

envp[] 要传递给新执行程序的完整的环境变量参数列表(可以为空).

Exec执行成功后将用一个新的程序代替原进程，但进程号不变，它绝不会再返回到调用进程了。如果exec调用失败，它会返回-1。

实验五 文件管理

实验目的

通过模拟文件的创建、删除操作；

加深对于操作系统文件管理功能的了解；

练习模拟算法的编程技巧，锻炼研究分析试验数据的能力。

实验说明

给出一个磁盘块序列：1、2、3、……、500，初始状态所有块为空的，每块的大小为2k。选择使用空闲表、空闲盘区链、位示图三种算法之一来管理空闲块。对于基于块的索引分配执行以下步骤：
（1） 随机生成2k-10k的文件50个，文件名为1.txt、2.txt、……、50.txt，按照上述算法存储到模拟磁盘中。
（2） 删除奇数.txt（1.txt、3.txt、……、49.txt）文件
（3） 新创建5个文件（A.txt、B.txt、C.txt、D.txt、E.txt），大小为：7k、5k、2k、9k、3.5k，按照与（1）相同的算法存储到模拟磁盘中。
（4） 给出文件A.txt、B.txt、C.txt、D.txt、E.txt的盘块存储状态和所有空闲区块的状态。

实验设计

1、基于位示图的存储空间管理

​ 每个二进制位对应一个磁盘块，一般“0”表示磁盘块空闲、“1“表示磁盘块已分配，这里为了方便显示，用”2“标记为索引块。

​ 一般用连续的字表示位示图，一个字的字长为16位，每一位表示一个磁盘块，因此可以用（字号，位号）对应一个磁盘块。500个磁盘块共需32个字，设计二维数组32*16表示位示图：

​ 磁盘块号从1开始，(字号,位号)从0开始，因此从磁盘块号到字号、位号存在一定的对应关系。

2、基于索引分配的文件分配方式

​ 索引分配允许文件离散地分配在各个磁盘块中，系统会为每个文件建立一个索引表，索引表中记录了文件的各个逻辑块对应的物理块。一个索引表占用一个物理块，称为索引块；而文件数据存放的物理块称为数据块。

​ 因此，需要在文件目录添加一个表项：索引块，存储该文件的索引表所存放的磁盘块号，访问文件数据时需要先把索引表存放的索引块调入内存。

3、索引分配+位示图

如何分配

（1）在目录添加一条记录，包括文件名、文件大小、索引块；

（2）顺序扫描位示图，找到一个空闲块作为该文件的索引块存放索引表信息，设置状态位为“2”，在目录中记录索引块号；

（3）若文件需要k个物理块，顺序扫描位示图找到k个相邻或不相邻的空闲块；

（4）根据字号、位号算出对应的磁盘块号，记录在索引表中，将相应的磁盘块分配给文件；

（5）将位示图相应位设置为“1”，已分配数据块。

如何回收

（1）根据目录找到索引块存放位置，访问索引表，找到文件索引表；

（2）根据索引表提供的文件数据存放的磁盘块号，计算出位示图对应的字号、位号；

（3）设置位示图相应位为“0”，回收磁盘块；

（4）设置索引块对应的位示图状态为“2”，删除索引表，在目录删除文件记录。

实验结果

（1） 随机生成2k-10k的文件50个，文件名为1.txt、2.txt、……、50.txt，按照上述算法存储到模拟磁盘中。

（2） 删除奇数.txt（1.txt、3.txt、……、49.txt）文件

（3） 新创建5个文件（A.txt、B.txt、C.txt、D.txt、E.txt），大小为：7k、5k、2k、9k、3.5k，按照与（1）相同的算法存储到模拟磁盘中。

（4） 给出文件A.txt、B.txt、C.txt、D.txt、E.txt的盘块存储状态和所有空闲区块的状态。
